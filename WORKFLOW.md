# AI 自媒体内容生成工作流

## 指令一览

| 指令 | 说明 |
|-----|------|
| `/idea` | 推荐5个下一篇主题（基于往期内容+整体目标） |
| `/daily 日期 主题：xxx` | 生成指定日期的全部内容（文章+配图） |

---

## 流程一：主题推荐 (`/idea`)

### 触发方式
```
/idea
```

### 执行步骤
1. 读取 `TOPICS.md` 获取：
   - 最终目标（365天计划的整体方向）
   - 已发布的主题列表
   - 主题分类和覆盖情况
2. 分析：
   - 哪些分类还没覆盖
   - 哪些是热点话题
   - 什么主题适合当前阶段
3. 输出5个推荐主题，按优先级排序，每个包含：
   - 主题名称
   - 推荐理由
   - 适合的发布时机

### 输出示例
```
📋 下一篇主题推荐（按优先级排序）

1. 【Cursor 使用技巧入门】
   理由：AI编程工具是热门话题，且尚未覆盖开发工具类
   时机：适合工作日发布，开发者活跃

2. 【用AI写周报，10分钟搞定】
   理由：职场实用技巧，受众广，之前的工具盘点可以延伸
   时机：周四/周五发布效果好

3. ...
```

## 流程二：内容生成 (`/daily`)

### 触发方式
```

```

或自然语言：
```
帮我生成2026年1月3日的内容，主题是"Cursor使用技巧入门"
```

## 内容生成执行流程

### 步骤1：生成三平台文章

按顺序生成以下文章，每篇文章末尾包含配图提示词：

| 平台 | 文件名格式 | 配图数量 | 图片比例 |
|-----|-----------|---------|---------|
| 小红书 | `日期-xiaohongshu-标题.md` | 6张 | 3:4 竖版 |
| 公众号 | `日期-wechat-标题.md` | 4张 | 16:9 横版 |
| 掘金 | `日期-juejin-标题.md` | 3张 | 16:9 横版 |

保存位置：`content/articles/`

### 步骤2：用 MCP Playwright 生成配图

1. 打开 Gemini (https://gemini.google.com)
2. 逐张输入提示词生成图片
3. 下载图片
4. 整理到 `content/assets/日期/` 目录
5. 按平台和序号重命名

### 步骤3：返回完成报告

```
✅ 2026-01-03 内容生成完成

文章：
- content/articles/2026-01-03-xiaohongshu-xxx.md
- content/articles/2026-01-03-wechat-xxx.md
- content/articles/2026-01-03-juejin-xxx.md

配图：
- content/assets/2026-01-03/ (13张)

请检查后发布。
```

## 质量要求

### ⚠️ 核心原则：图片与文章内容必须匹配

**错误示例：**
- 文章讲国产AI（豆包、通义、Kimi）
- 图片却出现 ChatGPT、Claude 等国外工具

**正确做法：**
- 提示词中明确指定文章提到的具体工具名称
- 如文章讲豆包，图片提示词要包含"豆包"、"Doubao"等关键词
- 避免使用通用的"AI工具"描述

### 配图提示词规范

1. **具体化**：提及文章中的具体工具/概念名称
2. **中文元素**：需要时包含中文文字
3. **风格统一**：扁平插画风格，蓝紫色渐变主题
4. **比例正确**：
   - 小红书：竖版 3:4
   - 公众号/掘金：横版 16:9

### 提示词模板

```
小红书封面图：
[具体工具名称]的介绍卡片，突出[核心特点]，扁平插画风格，蓝紫色渐变，竖版3:4比例，包含中文标题"[标题文字]"

公众号/掘金配图：
[场景描述]，展示[具体工具名称]的[功能/特点]，扁平插画风格，科技感，横版16:9比例
```

## 平台特点参考

### 小红书
- 风格：轻松、实用、种草
- 标题：吸引眼球，可用emoji
- 图片：精美、有设计感、适合滑动浏览
- 字数：500-1000字

### 公众号
- 风格：深度、专业、有价值
- 标题：清晰、有信息量
- 图片：配合文章节奏，辅助理解
- 字数：1500-2500字

### 掘金
- 风格：技术向、有代码、有干货
- 标题：直接、技术关键词
- 图片：流程图、对比图、代码截图
- 字数：1500-3000字

---

## 注意事项

1. **日期格式**：统一使用 `YYYY-MM-DD`
2. **文件命名**：使用英文和连字符，避免空格
3. **图片格式**：PNG 格式，确保清晰度
4. **先检查再发布**：生成完成后人工检查一遍内容质量
