# /idea - 主题推荐

当用户输入 `/idea` 时，执行以下任务：

## 执行步骤

1. **读取主题管理文件**
   - 读取 `TOPICS.md` 获取：
     - 最终目标和内容定位
     - 主题分类及覆盖状态
     - 已发布主题列表
     - 待选主题池

2. **读取已有文章**
   - 扫描 `content/articles/` 目录
   - 了解已发布的内容主题和风格

3. **分析并推荐**
   - 识别尚未覆盖的分类
   - 考虑内容的连贯性和进阶性
   - 结合当前时间点的热点话题
   - 平衡三类受众：小白用户、职场人士、开发者

4. **输出5个推荐主题**，按优先级排序，每个包含：
   - 主题名称
   - 所属分类
   - 推荐理由（为什么现在适合发这个）
   - 发布时机建议

## 输出格式

```
📋 下一篇主题推荐（按优先级排序）

1. 【主题名称】⭐ 强烈推荐（如果是第一优先）
   - 分类：xxx
   - 理由：xxx
   - 时机：xxx

2. 【主题名称】
   - 分类：xxx
   - 理由：xxx
   - 时机：xxx

...（共5个）

我的建议：先发 #X xxx，因为 xxx。

选哪个？或者你有其他想法？
```

## 注意事项

- 推荐的主题要与已发布内容形成递进关系
- 避免重复已发布的主题
- 优先推荐待选主题池中的高优先级主题
- 可根据节日、热点调整优先级
